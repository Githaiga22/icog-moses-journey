What I Learned:

**MOSES Terminology (complete set):**
- Program: a Combo tree of operators/variables/values — arguments written as #1, #2, ..., #n
- Exemplar: the current best program — MOSES always builds from the exemplar, never from scratch
- Representation: exemplar + knobs attached at various positions in the tree
- Knob: a single tunable parameter — discrete (fixed settings) or continuous (numeric range)
- Instance: one specific set of knob settings packed into a bit-string
- Field set: the schema mapping bits in an instance to specific knob settings
- Knob mapping: connects field_set entries to actual knob positions in the combo_tree
- Neighborhood: instances differing by 1 or 2 knob settings (Hamming distance)
- Deme: a population of instances all built around ONE representation
- Metapopulation: the collection of ALL demes, sorted by score — the outer world
- Scoring function: the judge — perfect score = 0, worse scores are negative
- Domination: Program A dominates B if A beats B on EVERY test row — otherwise both survive
- Normalization/reduction: simplifying programs using rewriting rules to elegant normal form

**The 5-step MOSES algorithm:**
1. Selection — pick best unexplored exemplar from metapopulation
2. Representation-building — attach knobs to exemplar, build field_set + knob_mapper + initial instance
3. Optimization (inner loop) — score instances, generate neighbors, score them, keep best (hill-climbing / simulated annealing / BOA)
4. Close deme — convert best instances back to programs, normalize, merge into metapopulation (drop dominated)
5. Repeat until perfect score or evaluation limit

**Key insight:** MOSES has two nested loops:
- Inner loop (local search): optimizer works within one deme
- Outer loop (global search): metapopulation manages competition between demes

**Why MOSES beats standard genetic programming:**
- Normalization removes bloat (spaghetti code)
- BOA finds dependencies between program parts — avoids pointless mutations
- Smaller, faster, more readable programs result

**Normalization examples from the guide:**
- `#3 OR False` reduces to `#3`
- `0 < 0.5 * #6` reduces to `0 < #6`
- `if (x = x) then y` reduces to `y`
